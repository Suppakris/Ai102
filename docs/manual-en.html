1. Project Overview

Ai102 is a local-first AI presentation generator, a customized build derived from ALLWEONE's open-source presentation-ai. Text generation runs on a local (or tunneled) Ollama model by default — no cloud LLM subscription required — with OpenRouter available as an optional, admin-funded upgrade.

The app takes a topic (or a source document: PDF, Word, Excel, CSV, or plain text), generates an editable outline, then produces a full slide deck with free AI-generated images, editable themes, rich text, charts, and PowerPoint export.

The headline feature on top of generation is the AI Deck Review: an auditor model scores every deck on clarity, design, and content accuracy, fact-checks its claims, and — with one click — auto-fixes failing decks by rewriting the flagged slides (with undo).

Sign-in is real OAuth (GitHub required; Google and Discord optional). All core features are free for every signed-in user.

Since V2.1 (2026-07-21)

A day of live bug-fixing closed out most of what V2.1's Known Issues section flagged. Highlights:

  • Uploading your own images into slides now works end-to-end (self-hosted storage, wired into the background/image editing panels) — no longer a known limitation.
  • Fixed the root causes of unreliable multi-slide generation: batches losing shared context (so a 12-slide outline could come back as ~6, deduplicated) and silent stalls after an early batch failure now retry once and surface a real error instead of quietly giving up.
  • Present mode: fixed exiting wiping/regenerating the deck, images not filling/centering the screen, animation glitches and page bounce, and a freeze when changing the presentation theme.
  • Fixed a Google sign-in failure for accounts that had already signed up via GitHub (account linking was previously refused).
  • AI Deck Review now automatically falls back to a free OpenRouter model when the shared Ollama tunnel is unreachable, flagged in the UI so you know a review ran on a backup model.
  • Editor stability: fixed freezing, selections jumping on their own, and Slate crashes during undo/redo on larger decks.
  • UI simplification: the right-edge insertion rail is back to 4 always-visible buttons (Text/Elements/Charts/Media embed); the redundant infographic data-editor dialog was removed in favor of the existing "Change template" flow.

3. What's Different From Upstream

  • Auth is real OAuth (src/backend/auth.ts, Auth.js/NextAuth v5 + Prisma). GitHub is required; Google and Discord are optional — each provider's sign-in button renders only when its credentials are configured. All three are free. New sign-ins default to role: USER.
    Test social sign-in only on the real deployed domain, never a Vercel PR preview URL. OAuth apps support exactly one callback URL (production), so preview-URL sign-ins always fail with error=Configuration. Expected, not a bug.
  • Text generation defaults to Ollama, with OpenRouter as an optional paid upgrade (src/lib/model-picker.ts). Requests resolve to a model served from OLLAMA_BASE_URL's native API unless OPENROUTER_API_KEY is set, which unlocks 6 preset cloud models for all users (the key's owner pays per token).
  • Image generation is free by default — Pollinations.ai (no API key) everywhere images are generated; FAL (Flux) is an optional admin-gated paid upgrade.
  • Backend logic lives under src/backend/ — db, auth, tenants, rate limiting, the image queue, and the LangGraph agent. See ARCHITECTURE.md.

4. Features

Core Functionality

  • AI outline generation, then full slide generation, on a local Ollama model
  • Use your own documents as source material: drop a PDF, Word (.docx), Excel (.xlsx/.xls), CSV, or text/Markdown file into the prompt box — parsed entirely in the browser (never uploaded), trimmed to the model's context budget
  • Editable outlines · real-time generation · auto-save
  • In-editor chat agent (LangGraph + Postgres memory): edit layout/background, replace/regenerate images, create/delete slides, switch themes, web search (needs TAVILY_API_KEY)

AI Deck Review & Auto-fix HEADLINE FEATURE

  • One-click Review in the editor header: scores 0–10 on clarity, design, content accuracy, with a pass / needs-revision verdict
  • Claim audit: every verifiable factual claim is marked Supported / Unsupported / Unverifiable — the presentation's own prompt and outline are passed as source context, so claims that trace back to them are genuinely verified
  • Auto-fix: a failing deck gets one corrective AI pass — unsupported claims removed or softened (never given invented backing), the rewrite is re-reviewed and applied to the editor, with one-click Undo. All-or-nothing: a rewrite that can't be parsed back into slides leaves the deck untouched
  • Feedback in the deck's own language (Thai decks get Thai feedback); sparse decks get clarifying questions instead of blind scores
  • Backend: reviewSlides() / reviewAndRevise() in src/backend/ai/reviewSlides.ts, served by POST /api/presentation/review-deck (session auth + rate limit), schema-enforced JSON output
  • Test harnesses: pnpm review:test and pnpm review:bash (adversarial decks)

Design & Customization

  • Built-in themes, custom theme creation, PPTX theme import; picking a theme no longer recolors the page chrome

Presentation Tools

  • Present from the app · public share links · PPTX export · charts/infographics/embeds · Plate rich-text editor · inline comments

Images

  • Pollinations.ai (free, default) · FAL Flux (optional paid) · Unsplash/Pixabay/Giphy stock · Google Custom Search

6. Getting Started

Prerequisites

  • pnpm (pinned to pnpm@11.1.3)
  • A PostgreSQL database (Supabase, Neon, or the Docker Postgres from Section 9)
  • Ollama running locally or reachable via OLLAMA_BASE_URL, with a model pulled (e.g. ollama pull qwen2.5:7b — the review feature's reference model)
  • GitHub OAuth app credentials (free) for sign-in

Installation

git clone https://github.com/Suppakris/Ai102.git
cd Ai102
pnpm install                  # postinstall runs prisma generate
cp .env.example .env          # fill in DATABASE_URL + auth vars (Section 7)
pnpm db:migrate:deploy        # apply SQL migrations (Section 8)
ollama pull qwen2.5:7b
pnpm dev

The app runs at http://localhost:3000. Signed-out visitors see the landing page; sign in with GitHub (or Google/Discord if configured) to reach the /presentation dashboard.

7. Environment Variables

.env.example is the source of truth. The notable ones:

VariableRequired?Purpose
DATABASE_URLRequiredPostgres connection string. For Supabase use the Transaction pooler (port 6543) — serverless functions exhaust Session/Direct pooler connections fast.
NEXTAUTH_SECRET / NEXTAUTH_URLRequiredAuth.js session signing + canonical URL.
GITHUB_CLIENT_ID / GITHUB_CLIENT_SECRETRequiredGitHub OAuth app (free). The app won't boot in production without auth vars.

Admin access

New sign-ins default to role: USER. To grant admin (paid FAL models, editing system themes):

pnpm db:studio    # open the User table, set role = ADMIN
-- or SQL:
UPDATE "User" SET role = 'ADMIN' WHERE email = 'their@email.com';

8. Database Setup (SQL Migrations)

The schema is managed with committed SQL migrations in prisma/migrations/:

pnpm db:migrate:deploy   # apply all pending migrations
pnpm db:migrate:dev      # after editing schema.prisma: generate + apply a new migration
pnpm db:studio           # browse the DB

Commit each generated folder — the SQL file is the schema change. pnpm db:push exists for throwaway prototyping only; never use it on a shared database.

Current migrations: 0_init → 20260714110000_add_slide_audit → 20260718150000_add_tenancy_and_agent_threads.

Existing databases created with db push (including the production Supabase DB) already contain all tables. Baseline them once instead of re-applying — otherwise migrate deploy will try to create tables that already exist: 
pnpm exec prisma migrate resolve --applied 0_init
pnpm exec prisma migrate resolve --applied 20260714110000_add_slide_audit
pnpm exec prisma migrate resolve --applied 20260718150000_add_tenancy_and_agent_threads
After baselining, pnpm db:migrate:deploy reports "No pending migrations" and future migrations apply normally.

No seed step required — signing in creates the user record via the Prisma adapter.

9. Running on Docker

The whole stack — front-end + back-end + Postgres + Redis + the image-generation worker — runs on Docker with nothing on the host except Docker itself:

docker compose up --build

Boot order is automatic: Postgres becomes healthy → a one-shot migrate container runs prisma migrate deploy → the app starts at http://localhost:3000 alongside a separate BullMQ worker container consuming the Redis-backed image queue.

LLM access from inside the container

  • Ollama on the host (default): compose points OLLAMA_BASE_URL at host.docker.internal:11434.
  • OpenRouter: put OPENROUTER_API_KEY=sk-or-... in a .env next to docker-compose.yml. (This path is now fully wired into the app — V1.0's "dead code" warning no longer applies.)

Auth inside Docker

Set real GITHUB_CLIENT_ID/GITHUB_CLIENT_SECRET (and optionally the Google/Discord pairs) in the same .env. With the placeholder defaults the stack boots and serves pages, but sign-in won't work.

Postgres data persists in the db-data volume; docker compose down -v wipes it.

11. Usage Guide

  1. Sign in at the deployed site (or pnpm dev locally) with GitHub, Google, or Discord.
  2. Create a deck from the dashboard: describe a topic in the prompt box — or drop in a PDF/Word/Excel/CSV/text file as source material — pick slide count, language, and theme, then generate. Review and edit the outline before rendering slides.
  3. Edit in the editor: change themes (the ⌄ next to the title opens the document menu; ☰ opens app navigation), edit text directly, or ask the chat agent for changes.
  4. Review: click Review in the header. You get scores, written feedback, and a claim audit. If the deck fails, click Auto-fix deck — the AI rewrites flagged slides and re-reviews; use Undo if you don't like the result. A deck built on fabricated claims may honestly stay "needs revision" — the AI removes fake claims but cannot invent real data.
  5. Ship it: export to .pptx, present from the browser, or share a public link.
If the header shows "Backend issue detected", the Ollama server is unreachable — usually the ngrok tunnel restarted and OLLAMA_BASE_URL needs the new URL (then redeploy).

13. Known Issues

  • Slide generation can occasionally glitch — deck content is written live by an AI model, so a run can still occasionally produce a broken layout or a half-filled slide. This is not data loss: regenerating the slide (or the whole deck) fixes it. The concrete causes we could name — batches losing shared context and dropping/duplicating slides, generation silently stopping after a failed batch — were fixed 2026-07-21; what remains is ordinary AI-generation variability, not a known defect.
  • Document/RAG search is scaffolded, not implemented — no vector search is wired into the agent; "chat with your document" doesn't work yet. (Attaching documents as generation source material does work — that's a different, shipped path.)
  • New sign-ins default to role: USER — FAL paid models and system-theme editing stay hidden until admin is granted (Section 7). All core features work at USER.
  • Review history is not persisted — reviews display in the dialog only; a stored history is the agreed next feature.
  • Shared-Ollama single-file line — one Ollama server serializes concurrent generation requests; heavy simultaneous use queues up. AI Deck Review now automatically falls back to a free OpenRouter model when the shared Ollama tunnel is unreachable (flagged in the UI as having run on a backup model) — deck generation itself still needs the model picker switched to OpenRouter by hand if Ollama is down.
  • Legacy .doc (Word 97–2003) is not supported as source material (modern .docx only); scanned PDFs need selectable text.
  • Both Biome and Prettier configs exist; Biome is canonical.
